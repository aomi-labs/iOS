# Aomi iOS

AI-powered crypto wallet assistant iOS app.

## Project Structure

```
Aomi/                       # XcodeGen project root
  project.yml               # XcodeGen project spec (source of truth)
  Aomi/                     # App source
    App/                    # AomiApp entry point, AppConfig
    Models/                 # API types, persistence (SwiftData)
    Services/               # ParaWalletService, AomiAPIClient, KeychainService
    ViewModels/             # AuthViewModel, ChatViewModel, etc.
    Views/
      Chat/                 # Chat UI, message views, widgets
      Common/               # Shared UI components (AuthStyles)
      Onboarding/           # AuthLoginView, AuthVerifyOTPView, WelcomeView
      SessionList/          # Session list, row views
      Wallet/               # WalletManagementSheet, WalletRowView
    Info.plist              # Manually maintained (not regenerated by XcodeGen)
    Aomi.entitlements       # Associated domains for Para
  AomiTests/                # Unit tests
ExportOptions.plist         # App Store Connect export config
```

## Tech Stack

- **Swift 6.0**, iOS 17.0+, SwiftUI
- **XcodeGen** for project generation (`project.yml` -> `.xcodeproj`)
- **Para Swift SDK 2.6.0** (`ParaSwift`) - embedded wallet auth & signing
- **MarkdownUI** - markdown rendering in chat
- **SwiftData** - local persistence (sessions, watch addresses)
- **@Observable** pattern (not ObservableObject/Combine)

## Key Commands

```bash
# Generate Xcode project (run after changing project.yml)
cd Aomi && xcodegen generate

# Build for simulator
xcodebuild -project Aomi.xcodeproj -scheme Aomi -sdk iphonesimulator -destination 'platform=iOS Simulator,name=iPhone 16 Pro' build

# Build release archive
xcodebuild -project Aomi.xcodeproj -scheme Aomi -configuration Release -archivePath /tmp/Aomi.xcarchive archive -allowProvisioningUpdates

# Export IPA
xcodebuild -exportArchive -archivePath /tmp/Aomi.xcarchive -exportOptionsPlist ExportOptions.plist -exportPath /tmp/AomiExport

# Upload to TestFlight (requires asc CLI configured)
asc builds upload --app "6759941797" --ipa "/tmp/AomiExport/Aomi.ipa"

# Run tests
xcodebuild -project Aomi.xcodeproj -scheme AomiTests -sdk iphonesimulator -destination 'platform=iOS Simulator,name=iPhone 16 Pro' test
```

## Para SDK Integration

**Reference docs:** https://docs.getpara.com/llms-full.txt (fetch this for Para API/SDK context)

### Configuration
- Environment: `beta` (via `AppConfig.paraEnvironment`)
- API Key: stored in `AppConfig.paraAPIKey` (beta key, 50 user limit)
- App scheme: `aomi` (registered as URL scheme in Info.plist)
- Dashboard: https://app.beta.getpara.com

### Auth Flow
1. `ParaWalletService.initiateAuth()` -> `paraManager.initiateAuthFlow()`
2. If new user: show OTP screen, user enters code
3. `handleVerificationCode()` -> `completeAuth()` (handleSignup or handleLogin)
4. Post-auth: fetch wallets, create default EVM wallet if none

### Critical SDK Constraints
- **Must call `paraManager.setDefaultWebAuthenticationSession(webAuthenticationSession)`** before auth flow (done in `AuthViewModel.init`)
- **Para SDK does NOT support concurrent operations** - all calls must be sequential
- **Simulator uses `.password` method** (passkeys don't work); device uses `.passkey`
- **Beta test emails**: use `<anything>@test.getpara.com`, any 6-digit OTP works
- URL scheme `paraswift` must be registered (required by SDK internally)

### Associated Domains
Entitlements include `webcredentials:app.usecapsule.com` and `webcredentials:app.beta.usecapsule.com` for Para passkey support.

## Architecture Notes

### Environment-based DI
Services are injected via SwiftUI environment:
```swift
.environment(apiClient)      // AomiAPIClient
.environment(walletService)  // ParaWalletService
```

### Active Wallet
- Users can switch between Para signing wallets and watch-only addresses
- Active wallet stored in `UserDefaults("activeWalletAddress")`
- `apiClient.publicKey` holds the current active address

### Info.plist
Manually maintained - NOT regenerated by XcodeGen. Uses build setting variables:
- `$(CURRENT_PROJECT_VERSION)` for CFBundleVersion
- `$(MARKETING_VERSION)` for CFBundleShortVersionString

Bump `CURRENT_PROJECT_VERSION` in `project.yml` before each TestFlight upload.
